# Chart Fix - All Recommendations Page

**Date**: 2026-03-03  
**Status**: ✅ Completed

## Problem

When clicking on the "All Recommendations" tab, the page went blank and the Instrument Mix chart failed to render.

## Root Cause

The `/api/v1/recommendations` endpoint (used for "All Recommendations" tab) was returning a simplified data structure that lacked required fields:
- Missing `instrument_mix` object (required by `InstrumentMixChart` component)
- Missing `agent_outputs` array (required by `ApprovalWorkflowCard` component)
- Missing other required fields like `id`, `run_id`, `agent_outputs`, etc.

The endpoint only returned minimal fields:
```json
{
  "recommendation_id": i,
  "status": "...",
  "created_at": "...",
  "optimal_hedge_ratio": 0.72,
  "expected_var_reduction": 0.42,
  "ifrs9_eligible": true
}
```

## Solution

Updated the `get_recommendations()` endpoint in `mock_backend.py` (lines 649-668) to return the complete data structure matching the `/api/v1/recommendations/pending` endpoint format:

### Changes Made:

1. **Added `instrument_mix` object**:
   ```python
   "instrument_mix": {
       "futures": futures_pct,
       "options": options_pct,
       "collars": collars_pct,
       "swaps": swaps_pct
   }
   ```

2. **Added `agent_outputs` array** with all 5 AI agents:
   - basis_risk
   - liquidity
   - operational
   - ifrs9
   - macro

3. **Added all required top-level fields**:
   - `id` (string format: `rec_0000000000000001`)
   - `run_id`
   - `expires_at`
   - `hedge_effectiveness_r2`
   - `collateral_impact`
   - `constraints_satisfied`
   - `action_required`
   - `risk_level`
   - `recommendation_text`

4. **Added `optimization_result` object**

5. **Added `approvals` array** (populated for APPROVED/REJECTED recommendations)

6. **Added `pages` field** to the response metadata for pagination
